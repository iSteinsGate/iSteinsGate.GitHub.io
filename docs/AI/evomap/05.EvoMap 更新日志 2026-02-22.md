# EvoMap 更新日志 2026-02-22：161 次提交，18 个新特性

**这一天，EvoMap 团队在 Hub 与 Website 两侧一共合入了 161 次提交，带来了 18 个重要更新。**

官方博客的这篇 Changelog 不是简单的「修修补补」，而是一次围绕「生物学隐喻 + 经济激励 + 搜索优先」的全栈升级。

## 提交概览

- **总计提交数**：161
  - Hub 仓库：68 次提交
  - Website 仓库：93 次提交
- **核心更新点**：18 项

下面是博客中提到的一些关键模块与方向。

## 一、Recipe & Organism 建模

EvoMap 把 Agent 能力抽象为类似「生物体」的结构：

- **Recipe（配方）**：描述如何构建一个具备特定能力的 Agent 组合；
- **Organism（个体）**：按照 Recipe 实际实例化出来的「智能体生命体」。

本次更新在这两块做了更精细的建模，包括但不限于：

- 更清晰的字段与类型设计；
- 更强的版本化与溯源能力；
- 为后续「能力市场」与「进化跟踪」打基础。

换句话说：  
> 过去我们只是「调用一个 Agent」，  
> 现在我们可以「设计、复制、进化一种 Agent 物种」。

## 二、Marketplace 统一

之前，EvoMap 生态中的各种资产——比如 Genes、Capsules、Recipes、Organisms——分布在不同视图与入口中。

这次更新做了一次 **Marketplace 统一**：

- 用一致的方式展示不同类型的「智能资产」；
- 更容易按能力、质量、使用次数等维度进行筛选；
- 为未来的「信用定价」「激励机制」预留了位置。

对于使用者来说，这意味着：

- 找能力像在商店里逛，而不是在一堆 Git 仓库里翻；
- 更容易发现「别人已经解决过的类似问题」，而不是重复造轮子。

## 三、接入 BoCha 搜索

更新日志提到，EvoMap 接入了 **BoCha 搜索**：

- 将 BoCha 作为底层搜索引擎之一；
- 用于在大规模 Gene / Capsule / 日志数据中做高质量检索；
- 尤其适用于「给定一个问题，找出历史上最相似的演化案例」这类任务。

这一步的本质是：  
让 EvoMap 从「一个存档系统」，变成「一个真正可搜索、可复用的进化记忆库」。

## 四、Biology Dashboard 扩展

所谓「Biology Dashboard」，可以理解为 EvoMap 的「生物学监控面板」：

- 展示当前网络中有哪些活跃的 Organisms；
- 哪些 Genes 在被频繁继承、哪些正在被淘汰；
- 不同演化路径的成功率、能耗与收益对比。

本次扩展包括：

- 更丰富的可视化指标；
- 更细粒度的过滤与分组；
- 更适合用来做「进化实验结果复盘」。

如果你习惯看 A/B Test 或实验平台的面板，可以把这里理解为：  
> 「面向 Agent 演化实验的生物学仪表盘」。

## 五、知识图谱（KG）搜索优先改版

在知识图谱（KG）相关的界面中，这次更新做了一个重要改动：  
**从「结构优先」切换为「搜索优先」。**

具体表现为：

- 把「搜索入口」放到更核心的位置；
- 让用户先通过搜索找到相关 Gene / Capsule / Organism，再去看其在图谱中的结构关联；
- 优化查询性能，让复杂查询也能在可接受的时间内完成。

这体现了一种很务实的产品哲学：  
> 在图再美之前，先让「找到东西」变得足够容易。

## 六、Economics 页面重构

EvoMap 不只是技术协议，也是一个带有激励与经济逻辑的网络。  
这次更新中，**Economics 页面**被大幅改版，重点包括：

- 更直观地展示 Credit 的流转路径；
- 清晰区分不同角色（贡献者、调用方、网络本身）的收益结构；
- 为未来的「代币化或记账体系」留出设计空间。

对贡献者来说，这意味着：

- 你更容易理解「为什么要贡献一个 Gene / Capsule」；
- 也更容易看到「自己的贡献是如何被调用、如何产生价值」。

## 七、UAT → Credit 全局重命名

更新中还提到了一项看似「小」，但影响范围很广的改动：  
**全局将 UAT 更名为 Credit。**

这意味着：

- 在界面、接口、文档中，统一使用 Credit 这一名称；
- 避免术语歧义，让新用户更快理解其含义（「可用于交换资源的积分」）；
- 为后续扩展 Credit 的用途（例如更多开发者资源、算力等）打基础。

## 八、Agent Network 的 viral growth

最后，Changelog 提到了一个非常有意思的点：  
**Agent Network 的「病毒式增长」。**

简而言之：

- 随着基因库的丰富，一个成功的 Gene 被更多 Agent 继承；
- 这些 Agent 在各自场景中再次迭代，又贡献新的 Gene 回到网络；
- 形成一个「解决方案 → 共享 → 继承 → 再演化 → 更强解决方案」的闭环。

从网络视角看，这是一次**正向的演化飞轮**：

> 越多 Agent 使用，网络就越聪明；  
> 网络越聪明，越多团队愿意接入；  
> 新团队带来的新场景，又催生新的进化。

## 小结：一次以「进化」为中心的版本更新

这篇 2026-02-22 的更新日志，核心并不是告诉你「修了几个 Bug」「加了几个按钮」，而是：

- 把 **Agent 视为生物体**（Recipe & Organism）；
- 把 **能力视为基因资产**（Marketplace 统一 + BoCha 搜索）；
- 把 **网络视为进化生态**（Biology Dashboard + Agent Network 增长）；
- 把 **贡献者视为经济参与者**（Economics 页面 + Credit 体系）。

如果你正在思考「下一代 AI 应用平台」应该长什么样，这份 Changelog 本身就是一份很有启发性的设计纲要。
