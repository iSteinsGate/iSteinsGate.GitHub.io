   └─ 将feature-list按priority排序
   Phase 2: 并行执行阶段 (多Agent、多会话)

   Loop until no pending tasks:
  1. 获取工作环境
     ├─ pwd + 确认工作目录
     ├─ 读取 progress-log.md（了解历史）
     └─ 读取 feature-list.json
  2. 领取任务
     ├─ 找到 status="pending" 且 priority最高
     ├─ 检查 agent-locks.json 是否可加锁
     ├─ 加锁任务（写入agent_id + expires_at）
     └─ 更新 status="in_progress"
  3. 实现任务
     ├─ 按steps列表逐步实现
     ├─ 编写代码 + 测试
     └─ 运行验证
  4. 验证质量
     ├─ 运行单元测试
     ├─ 运行集成测试
     └─ 端到端测试（如适用）
  5. 更新状态
     ├─ 如果测试通过：
     │  ├─ status → "passing"
     │  ├─ completed_at = now
     │  ├─ Git commit（描述性消息）
     │  └─ 更新 progress-log.md
     └─ 如果失败：
        ├─ status → "pending"（退回队列）
        ├─ attempts += 1
        └─ 记录 error_log
  6. 释放锁
     └─ 从 agent-locks.json 移除锁
  7. 会话结束检查
     └─ 检查是否还有pending任务

def claim_task(agent_id):
    feature_list = read_json("feature-list.json")
    locks = read_json("agent-locks.json")
    
    # 找到未锁定的高优先级任务
    for task in sort_by_priority(feature_list['features']):
        if task['status'] == 'pending':
            if task['id'] not in locks['locks']:
                # 加锁
                lock_task(task['id'], agent_id, expires_in=2_hours)
                task['status'] = 'in_progress'
                task['locked_by'] = agent_id
                save_json("feature-list.json", feature_list)
                return task
    
    return None  # 无可用任务
